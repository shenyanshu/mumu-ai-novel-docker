# MuMuAI 小说创作工具小白使用说明

这是一套 AI 小说创作系统。你可以用它创建小说项目，管理世界观、角色、剧情大纲、章节正文，并让 AI 辅助生成内容。

## 1. 准备工作

使用前先准备好：

1. 一台 Windows 电脑。
2. 已下载本系统代码或压缩包。
3. 如果要用 AI 生成功能，需要准备一个可用的 API Key，例如 OpenAI、中转 API 或其他兼容接口。

如果只是打开系统看看页面，可以先不填 API Key；如果要让 AI 写大纲、角色、章节，就必须配置 API Key。

## 2. 启动系统

### 方法一：本地一键启动

1. 打开项目文件夹。
2. 找到 `一键启动.bat`。
3. 双击运行。
4. 等待启动控制台出现。
5. 点击控制台里的“进入应用”，或用浏览器打开：

```text
http://127.0.0.1:8000
```

如果页面打不开，再试：

```text
http://localhost:8000
```

### 方法二：服务器或 Docker 启动

如果你是服务器部署，启动后通常打开：

```text
http://服务器IP:8000
```

本机 Docker 部署一般打开：

```text
http://localhost:8000
```

### 启动前：数据库怎么配置

本系统使用 SQLite 嵌入式数据库，无需安装任何数据库软件。首次启动时自动创建。

如果你是 Docker 部署，按这个做：

```powershell
Copy-Item .env.example .env
New-Item -ItemType Directory -Force secrets | Out-Null
Set-Content -Path secrets/local_auth_password.txt -Value "这里换成登录密码"
docker compose -f docker-compose.yml -f docker-compose.prod.yml pull
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

如果你是本地一键启动或使用打包版，数据库文件会自动创建在 `backend/data/mumuai.db`。无需额外配置。

如需自定义数据库路径，一键启动/打包版改 `config.ini`，Docker 部署改 `.env`：

```text
DATABASE_URL=sqlite+aiosqlite:///./data/mumuai.db
```

### 启动前：向量模型放哪里

向量模型不是数据库，它是本地离线模型文件。必须放在：

```text
backend/embedding/models--sentence-transformers--paraphrase-multilingual-MiniLM-L12-v2
```

这个目录下面应该能看到类似 `snapshots`、`refs`、`blobs` 这样的子目录。

Docker 启动后，容器里会检查这个路径：

```text
/app/embedding/models--sentence-transformers--paraphrase-multilingual-MiniLM-L12-v2
```

GHCR 预构建镜像已经内置离线向量模型，普通 Docker 部署不需要手动准备 `backend/embedding/...`。

如果你要自己构建镜像，构建过程会下载并缓存模型到 `/app/embedding`，不需要把模型目录提交到 GitHub。

向量数据库生成的数据在：

```text
本地一键启动：backend/data/chroma_db
Docker 部署：/app/data/chroma_db
```

Docker 部署时数据会存在 Docker volume 里，不要把它当成模型文件。

## 3. 登录系统

系统目前使用本地账号登录。

默认本地账号通常是：

```text
账号：admin
密码：由部署配置决定
```

如果你是 Docker 部署，密码一般写在：

```text
secrets/local_auth_password.txt
```

如果你是本地使用，密码一般看：

```text
一键启动/打包版：config.ini
直接运行后端开发服务：backend/.env
```

登录成功后，会进入“项目书架”。

## 4. 配置 AI

第一次使用 AI 前，先配置模型。

操作步骤：

1. 登录系统。
2. 点击右上角或菜单里的“设置”。
3. 在“API 配置”里选择提供商。
4. 填写 `API Base URL`。
5. 填写 `API Key`。
6. 点击“测试连接”。
7. 测试成功后，点击“刷新模型列表”。
8. 选择要使用的模型。
9. 点击“保存设置”。

常见填写方式：

```text
API 提供商：OpenAI
API Base URL：https://api.openai.com/v1
API Key：你的 sk-xxx 密钥
模型：选择可用模型
```

如果你用的是中转接口：

```text
API 提供商：OpenAI 或 自定义
API Base URL：中转平台给你的地址
API Key：中转平台给你的密钥
模型：中转平台支持的模型名
```

只要“测试连接”成功，就说明 AI 配置可用。

## 5. 创建第一本小说

进入系统后，你会看到“项目书架”。

有两种创建方式：

### 快速新建

适合你已经想好书名，只想先建一个空项目。

操作：

1. 点击“快速新建”。
2. 填写书名。
3. 可选填写简介、主题、类型、目标字数、章节数、角色数。
4. 点击“直接创建”。
5. 创建完成后进入项目。

### AI 生成项目

适合你想让 AI 帮你生成世界观、角色和大纲。

操作：

1. 点击“快速新建”。
2. 填写书名和简介。
3. 选择类型，例如玄幻、都市、科幻等。
4. 选择叙事视角，例如第一人称、第三人称。
5. 填写目标字数、章节数、角色数。
6. 在“额外要求”里写清楚你想要什么。
7. 点击“AI 生成”。
8. 等待系统自动生成世界观、角色和故事大纲。
9. 生成完成后点击“进入项目”。

额外要求示例：

```text
主角从底层开始成长，前期节奏爽快，中期加入宗门斗争，后期揭开世界真相。女主不要太早出现，反派要有压迫感。
```

## 6. 项目工作台怎么用

进入项目后，左侧是功能菜单，主要包括：

```text
世界设定
世界规则
角色与组织
关系管理
故事大纲
章节管理
剧情分析
写作风格
记忆系统
```

推荐使用顺序：

```text
世界设定 -> 角色与组织 -> 故事大纲 -> 章节管理 -> 剧情分析 -> 继续优化
```

## 7. 世界设定

这里用来管理小说的基础世界观。

主要内容：

```text
时代背景
地点设定
氛围基调
世界规则
```

常用操作：

1. 点击“编辑内容”，手动修改设定。
2. 点击“保存修改”，保存当前内容。
3. 点击“重新生成”，让 AI 根据项目资料重新生成世界设定。

注意：“重新生成”可能覆盖当前内容。如果你写了很重要的设定，先自己备份一下。

## 8. 角色与组织

这里用来管理小说人物、势力、宗门、公司、家族等。

### 添加角色

1. 点击“添加角色”。
2. 填写角色名字。
3. 选择角色类型，例如主角、配角、反派。
4. 填写性格特点和背景故事。
5. 点击“保存”。

### AI 生成角色

1. 点击“AI 生成”。
2. 可填写角色名称，也可以留空让 AI 起名。
3. 选择角色定位。
4. 填写背景设定或额外要求。
5. 点击“开始生成”。

### 添加组织

1. 点击“添加组织”。
2. 填写组织名称。
3. 填写组织类型，例如宗门、家族、商会、公司。
4. 填写组织宗旨、背景和风格。
5. 保存。

### AI 生成组织

1. 点击“AI 生成组织”。
2. 填写组织要求。
3. 点击“开始生成”。

示例：

```text
生成一个掌控北方雪山的修仙宗门，门规严厉，内部有长老派系斗争，表面正道，暗中研究禁术。
```

## 9. 故事大纲

“故事大纲”页面有多个页签：

```text
故事大纲
剧情卡片
剧情线
章纲
关联总览
```

建议先生成“故事大纲”，再生成“章纲”。

### 故事大纲

用途：确定整本书的主线方向。

操作：

1. 点击“AI 生成”。
2. 设置叙事视角、章节数、目标字数。
3. 填写额外要求。
4. 点击开始生成。

### 剧情卡片

用途：记录一个个重要剧情点。

例子：

```text
主角第一次觉醒能力
女主身份暴露
反派第一次正面压制主角
宗门大比前夜有人失踪
```

### 剧情线

用途：管理主线、支线、角色线、伏笔线。

例子：

```text
主线：主角追查父亲失踪真相
支线：宗门内部派系争斗
角色线：女主从敌对到信任主角
伏笔线：禁术来源和上古遗迹
```

### 章纲

用途：把整本书拆成每一章要写什么。

操作：

1. 进入“章纲”页签。
2. 可以手动新建章纲。
3. 也可以点击 AI 生成章纲。
4. 每个章纲可以写章节标题、场景、视角角色、剧情要点、目标字数。

## 10. 章节管理

这里是真正写正文的地方。

### 从章纲同步章节

如果你已经生成了章纲：

1. 进入“章节管理”。
2. 点击“从章纲同步”。
3. 系统会根据章纲创建章节列表。

### 手动新建章节

1. 点击“手动新建”或“新建章节”。
2. 填写章节标题和章节序号。
3. 可以直接输入正文。
4. 点击保存。

### AI 生成单章

1. 在章节列表找到要生成的章节。
2. 点击闪电图标或“AI 生成”按钮。
3. 设置目标字数。
4. 选择写作风格。
5. 是否启用 MCP 按需选择。
6. 点击“开始生成”。
7. 等待生成完成。

系统会自动参考：

```text
项目设定
章纲
前文内容
关联剧情卡片
角色信息
写作风格
```

### 重新生成章节

如果对某章不满意，可以点击重新生成。

注意：重新生成会覆盖原正文。确认前最好先复制原文备份。

### 批量生成章节

适合一次生成多章。

操作：

1. 点击“批量生成”。
2. 选择起始章节和结束章节。
3. 点击“开始生成”。
4. 系统会按顺序生成。
5. 已经有正文的章节会自动跳过，避免覆盖。

## 11. 沉浸式阅读

章节生成后，可以进入阅读模式。

在章节列表中打开章节阅读页后，可以：

1. 阅读当前章节正文。
2. 切换上一章、下一章。
3. 打开右侧章节分析。

章节分析可能包含：

```text
情节分析
角色状态
情感曲线
伏笔追踪
时间轴事件
关系变化
因果链
一致性审计
```

## 12. 写作风格

“写作风格”用来告诉 AI 你希望正文是什么味道。

比如：

```text
热血爽文
悬疑压迫感
轻松日常
古风细腻
快节奏网文
番茄风格短句
```

建议先创建一个常用风格，生成章节时选择它。

## 13. 记忆系统

记忆系统用于帮助 AI 记住长期设定，减少前后矛盾。

适合记录：

```text
主角能力限制
角色之间的秘密
已经埋下的伏笔
重要物品来历
世界规则禁忌
```

写长篇时建议经常维护记忆，不然 AI 容易忘设定。

## 14. 导入和导出

在项目书架页面，点击右上角“更多操作”。

你可以看到：

```text
导入项目
导出项目
导出 TXT
系统设置
```

### 导出项目

导出为 JSON 文件，适合备份或迁移。

### 导入项目

选择之前导出的 JSON 文件，系统会先验证文件格式，再导入。

### 导出 TXT

导出纯文本，适合发给别人阅读或继续排版。

## 15. 管理用户

如果你是管理员，可以进入“用户管理”。

可以做这些事：

```text
新建用户
设置管理员
禁用或启用用户
重置密码
删除用户
```

如果只是自己用，这一块可以不用管。

## 16. 推荐创作流程

新手按这个顺序用最稳：

```text
1. 登录系统
2. 进入设置，配置 API Key
3. 回到项目书架
4. 点击快速新建
5. 填书名、简介、类型、章节数
6. 点击 AI 生成项目
7. 进入项目后检查世界设定
8. 检查角色与组织
9. 进入故事大纲，补充章纲
10. 进入章节管理，从章纲同步章节
11. 单章生成或批量生成
12. 阅读章节，检查不满意的地方
13. 修改设定、角色、章纲后继续生成
14. 最后导出 TXT 或导出项目备份
```

## 17. 常见问题

### 页面打不开

先确认启动窗口没有报错。

再检查地址：

```text
本地一键启动：http://127.0.0.1:8000
Docker 部署：http://localhost:8000
前端 Vite 开发模式：http://localhost:5173
```

如果还是打不开，重启启动脚本。

### 登录失败

检查账号密码是否正确。

Docker 部署检查：

```text
secrets/local_auth_password.txt
```

本地使用检查：

```text
一键启动/打包版：config.ini
直接运行后端开发服务：backend/.env
```

### AI 不能生成

重点检查：

```text
API Key 是否填对
API Base URL 是否填对
模型是否选择
测试连接是否成功
账号余额是否够
网络是否能访问 API
```

### 生成很慢

可能原因：

```text
模型响应慢
章节目标字数太多
网络慢
同时生成任务太多
```

可以先把目标字数调小，例如 1500 到 3000 字。

### 生成内容不符合预期

优先改这几个地方：

```text
项目简介
世界设定
角色背景
章纲剧情要点
写作风格
额外要求
```

AI 不是只看一句提示，它会参考整个项目资料。资料越清楚，生成越稳定。

### 章节前后矛盾

建议：

```text
补充记忆系统
维护角色设定
把重要伏笔写进剧情线
每章生成后做章节分析
及时修正章纲
```

### 不小心生成错了

如果还没保存，可以关闭窗口。

如果已经覆盖，优先检查是否有导出的备份。重要章节重新生成前，建议先复制原文。

## 18. 给新手的一句话

不要一上来就直接批量生成全书。先把世界观、角色、大纲、章纲整理清楚，再从前几章开始试生成。前 3 章效果稳定后，再批量生成后面的章节。
